hexoblog的使用与书写

第一步：指向目标文件夹

cd /盘/my-blog例如：cd/d/Blog/my-blog

如果放在桌面，则不同：cd~/Desktop/my-blog

第二步：生成md文件

hexo new "你的文章标题"

#该阶段会出现的问题

  1. new与“间无空格，则无效果
  2. 创建路径不对

检查路径：hexo config | grep "post_dir“

第三步：使用markdown编辑器写文本（这里推荐typora）

  • 在my-blog/source/_post里找到刚才创建的md文件
  • 右键，选择打开方式，在typora文件夹里选择typora.exe
  • 打开md文件，开始写blog

注意：

  • 需要稍微了解下typora的使用

  • tepora简化了md文件的书写步骤，不会markdown也可

    书写blog

第四步：预览

hexo clean ——清理缓存

hexo serve或者hexo s ——预览

ctrl+c (快捷键) ——停止hexo s（预览结束后的步骤）

根据提示，去https://localhost4000/ 上预览自己写的md文件格式

若输入指令后，git bash出现大量黄色提示，这是在指出你的语法错误，需要根据提示修改md文件中的文本格式

第五步：投放到hexo上

hexo generate 或hexo g ——生成静态文件（存放在文件夹public中）

hexo deplay或 hexo d ——放置在deplay配置的线上仓库中（如：github）

若出现：INFO Deplay down git则成功

之后到网站 https://user.github.io 上便可以查看blog

（例如： https://cnkrru.github.io )

若未看到新的blog，则等待几分钟，刷新一下，部署过程可能慢了些

补充：更改/删除blog/报错后如何返回

更改：

在原来的md文件上之间修改，然后

hexo clean && hexo s ——预览一下

hexo g &&hexo d ——配置

删除：

在_post文件夹里将想要删除的blog所属的md文件删除

hexo clean && hexo s ——预览一下

hexo g &&hexo d ——配置

报错后如何返回：

ctrl+c ——中断当前步骤，返回至my-blog

不需要重新写md什么的，根据报错检查已有md的问题即可
